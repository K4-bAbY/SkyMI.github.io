# Contribution Guidelines

We warmly welcome and appreciate your contributions to the SkyMI Android Kernel project! Your participation is a vital driver for the project's continuous development. Before you start contributing, please take a few minutes to read this guide to ensure your contributions can be smoothly adopted.

## Ways to Contribute

You can contribute to the SkyMI Kernel project in several ways:

1.  **Code Contribution**: Submit kernel code patches, optimizations, or new features.
2.  **Documentation Improvement**: Improve existing documentation to make it clearer and more accurate.
3.  **Bug Reports**: Submit bugs you find during use.
4.  **Feature Suggestions**: Propose new features or improvement suggestions you believe are valuable.
5.  **Community Support**: Help other users solve problems in community forums or groups.

## Code Contribution Process

If you wish to contribute code, please follow these steps:

### 1. Fork the Repository

First, fork the SkyMI Kernel source code repository to your own GitHub account. If you are unsure of the source code repository address, please check the main `README.md` file or project release channels.

### 2. Clone Your Fork

Clone your forked repository to your local development environment:

```bash
git clone https://github.com/your-username/SkyMI-Kernel-Source.git
cd SkyMI-Kernel-Source
```

### 3. Create a New Branch

Create a new branch for your new feature or bug fix. Use a meaningful name, such as `feature/add-new-scheduler` or `fix/boot-loop-issue`.

```bash
git checkout -b feature/your-feature-name
```

### 4. Make Changes

Make code changes on your new branch. Ensure your code complies with the following specifications:

-   **Code Style**: Follow the Linux kernel coding style guidelines.
-   **Commit Messages**: Write clear, concise, and meaningful commit messages. The first line should be a short summary (less than 72 characters), followed by a more detailed description if necessary.
-   **Testing**: Thoroughly test your changes on your device to ensure stability and functionality.

### 5. Commit Changes

Commit your changes to the local repository:

```bash
git add .
git commit -m "feat: add your feature description"
```

### 6. Push to Remote Repository

Push your local branch to your forked remote repository:

```bash
git push origin feature/your-feature-name
```

### 7. Submit a Pull Request (PR)

Visit your forked GitHub repository page, and you will see a prompt to create a Pull Request. Fill in a detailed PR description, explaining your changes, purpose, and any related issues.

## Bug Reports

If you find a bug, please submit it via GitHub Issues. When submitting a bug report, provide as much detailed information as possible, including:

-   **Problem Description**: Clearly describe the issue you encountered.
-   **Reproduction Steps**: Provide detailed steps so we can reproduce the bug.
-   **Device Information**: Your device model, Android version, SkyMI Kernel version.
-   **Log Files**: If possible, attach relevant `logcat` or `dmesg` logs.

## Feature Suggestions

Welcome to submit feature suggestions via GitHub Issues. Please explain your idea in detail, including why you think this feature is valuable and how it will improve the user experience.
