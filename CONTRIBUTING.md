## 代码贡献流程

如果您希望贡献代码，请遵循以下步骤：

### 1. Fork 仓库

首先，将 SkyMI 内核的源代码仓库 Fork 到您自己的 GitHub 账户下。如果您不确定源代码仓库的地址，请查阅主 `README.md` 文件或项目发布渠道。

### 2. 克隆您的 Fork

将您 Fork 后的仓库克隆到本地开发环境：

```bash
git clone https://github.com/您的用户名/SkyMI-Kernel-Source.git
cd SkyMI-Kernel-Source
```

### 3. 创建新分支

为您的新功能或 Bug 修复创建一个新的分支。请使用有意义的名称，例如 `feature/add-new-scheduler` 或 `fix/boot-loop-issue`。

```bash
git checkout -b feature/your-feature-name
```

### 4. 进行更改

在您的新分支上进行代码更改。请确保您的代码符合以下规范：

-   **代码风格**：遵循 Linux 内核编码风格指南。
-   **提交信息**：编写清晰、简洁且有意义的提交信息。第一行应是简短的总结（少于 72 个字符），随后可以有更详细的描述。
-   **测试**：在您的设备上充分测试您的更改，确保其稳定性和功能性。

### 5. 提交更改

将您的更改提交到本地仓库：

```bash
git add .
git commit -m "feat: add your feature description"
```

### 6. 推送到远程仓库

将您的本地分支推送到您 Fork 的远程仓库：

```bash
git push origin feature/your-feature-name
```

### 7. 提交 Pull Request (PR)

访问您 Fork 的 GitHub 仓库页面，您会看到一个提示，引导您创建一个 Pull Request。请填写详细的 PR 描述，说明您的更改内容、目的以及任何相关的 Issue。

## Bug 报告

如果您发现 Bug，请通过 GitHub Issues 提交。在提交 Bug 报告时，请提供尽可能详细的信息，包括：

-   **问题描述**：清晰地描述您遇到的问题。
-   **复现步骤**：提供详细的步骤，以便我们能够复现该 Bug。
-   **设备信息**：您的设备型号、Android 版本、SkyMI 内核版本。
-   **日志文件**：如果可能，请附上相关的 `logcat` 或 `dmesg` 日志。
